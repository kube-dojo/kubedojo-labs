# Security Contexts

Security contexts are the primary mechanism for configuring pod and container security in Kubernetes (20% CKS weight).

## What You Will Learn

- Set runAsNonRoot and runAsUser to prevent root containers
- Enable readOnlyRootFilesystem to prevent filesystem modifications
- Control privilege escalation and Linux capabilities
- Apply security contexts to multi-container pods

# Control Plane Components

The Kubernetes control plane is the brain of the cluster. Understanding its components is a core CKA topic.

In this lab you will:

- Identify static pod manifests that define control plane components
- Explore etcd, the cluster's key-value store
- Inspect API server configuration flags
- Verify scheduler and controller manager health

**What you will learn:**
- How control plane components are deployed as static pods
- How to find etcd endpoints and configuration
- How to inspect API server flags like service-cluster-ip-range
- How to check control plane component health

**Time:** 40 minutes

**Curriculum module:** [CKA Part 1 - Control Plane](https://kube-dojo.github.io/k8s/cka/part1-cluster-architecture/module-1.1-control-plane/)

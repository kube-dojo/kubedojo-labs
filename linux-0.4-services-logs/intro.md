## Services and Logs

Modern Linux systems use **systemd** to manage services and **journald** to collect logs. Mastering both is essential for any sysadmin or DevOps engineer.

### What You'll Learn

- Manage services with `systemctl`: start, stop, enable, status
- Query logs with `journalctl`
- Create custom systemd unit files
- Analyze and rotate logs

If you haven't read the theory yet, check out [Module 0.4: Services and Logs](https://kube-dojo.github.io/linux/foundations/everyday-use/module-0.4-services-logs/) on KubeDojo.

## Networking Tools

Networking is at the heart of Kubernetes. Every pod, service, and ingress relies on Linux networking primitives. This lab builds your network troubleshooting toolkit.

### What You'll Learn

- Inspect interfaces, IPs, and routes with `ip`
- Resolve DNS with `dig`, `nslookup`, and `host`
- Test connectivity with `ping`, `curl`, and `nc`
- Monitor connections with `ss` and `netstat`

### Before You Start

If you haven't read the theory yet, check out [Module 0.5: Networking Tools](https://kube-dojo.github.io/linux/foundations/everyday-use/module-0.5-networking-tools/) on KubeDojo.

This lab takes about **35 minutes**. Let's go!

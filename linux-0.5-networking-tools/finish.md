## Lab Complete!

You now have a solid networking toolkit:

- **Interfaces**: `ip addr`, `ip link`, `ip route` — see your network config
- **DNS**: `dig`, `nslookup`, `host` — resolve names to IPs
- **Connectivity**: `ping`, `curl`, `nc` — test reachability and ports
- **Monitoring**: `ss`, `netstat` — see active connections and listeners

These are the same tools you'll use to debug Kubernetes networking: pod-to-pod, service resolution, ingress routing, and network policies.

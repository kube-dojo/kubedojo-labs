## Servers and SSH

Servers are the machines that run your applications. In cloud native infrastructure, you'll connect to them remotely using SSH, check what services are running, and verify data hasn't been tampered with.

This lab teaches you the practical skills you'll use daily when managing Kubernetes nodes, debugging networking issues, and securing access to remote systems.

### What You'll Learn

- Generate SSH key pairs with `ssh-keygen`
- Check listening ports with `ss`
- Read and understand SSH server configuration
- Verify file integrity with `md5sum`

### Before You Start

If you haven't read the theory yet, check out [Module 0.7: Servers and SSH](https://kube-dojo.github.io/prerequisites/zero-to-terminal/module-0.7-servers-ssh/) on KubeDojo.

This lab takes about **25 minutes**. Type the commands yourself -- muscle memory is the goal.

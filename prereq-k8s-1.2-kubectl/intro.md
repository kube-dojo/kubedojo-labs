## kubectl Basics

`kubectl` is the command-line tool for interacting with Kubernetes. It's the single most important tool you'll use — whether you're a developer, operator, or SRE.

### What You'll Learn

- List resources with `kubectl get` and various flags
- Inspect resources in detail with `kubectl describe`
- Use output formats: `-o wide`, `-o yaml`, `-o json`, `-o jsonpath`
- Generate manifests with `--dry-run=client` and explore the API with `kubectl explain`

### Before You Start

If you haven't read the theory yet, check out [Module 1.2: kubectl Basics](https://kube-dojo.github.io/prerequisites/kubernetes-basics/module-1.2-kubectl/) on KubeDojo.

This lab takes about **30 minutes**. Type the commands yourself — muscle memory is the goal.

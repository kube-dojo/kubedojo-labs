## Lab Complete!

You've mastered the essential kubectl commands:

- **Get**: `kubectl get` with `-A`, `-o wide`, `--no-headers` — list any resource type
- **Describe**: `kubectl describe` — deep inspection of any resource
- **Output formats**: `-o yaml`, `-o json`, `-o jsonpath` — extract exactly what you need
- **Dry run**: `--dry-run=client -o yaml` — generate manifests without applying them
- **Explain**: `kubectl explain` — built-in API documentation

These are the commands you'll use in every Kubernetes session. On the CKA/CKAD exams, `--dry-run=client -o yaml` alone saves enormous time.

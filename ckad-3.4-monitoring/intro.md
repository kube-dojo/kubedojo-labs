# Monitoring Applications

In this lab you will use kubectl to monitor resource usage and application health through metrics and events.

**What you will learn:**
- Use `kubectl top` for CPU and memory metrics
- Query cluster events for debugging
- Sort and filter pods by resource usage

**Time:** 25 minutes

**Curriculum module:** [CKAD Part 3 - Monitoring Applications](https://kube-dojo.github.io/k8s/ckad/part3-observability/module-3.4-monitoring/)

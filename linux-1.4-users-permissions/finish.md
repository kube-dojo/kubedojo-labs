## Lab Complete!

You've mastered Linux user management:

- **Users & groups**: `useradd`, `groupadd`, `usermod`, `id` — identity management
- **Passwords**: `/etc/shadow`, `passwd -l` — account security
- **sudo**: `/etc/sudoers`, per-command access — least privilege
- **Ownership**: `chown`, SGID directories — shared project access

These concepts map directly to Kubernetes: SecurityContext controls which user a container runs as, RBAC controls API access (similar to sudo), and volume permissions depend on understanding UIDs/GIDs.
